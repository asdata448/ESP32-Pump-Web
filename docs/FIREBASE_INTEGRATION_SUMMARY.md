## 📊 Cấu trúc dữ liệu Firestore

### Collection: `pump_history`

```typescript
interface PumpHistoryRecord {
  id?: string                              // Auto-generated by Firestore
  deviceId: string                         // Unique device ID
  syringeType: string                      // "10CC" or "20CC"
  speedMlh: number                         // ml/h
  volumeMl: number                         // Target volume in ml
  infusedVolumeMl: number                  // Actual infused volume
  totalTimeSec: number                     // Elapsed time in seconds
  status: 'COMPLETED' | 'STOPPED' | 'ERROR'
  timestamp: Timestamp                     // Firestore timestamp
  createdAt?: Timestamp                    // Server timestamp
  deviceIdString?: string                  // Device ID string
  notes?: string                           // Optional notes
  errorType?: string                       // Error type if any
}
```

## 🔧 Các tính năng chính

### Auto-save
- Tự động lưu khi session DONE (hoàn thành)
- Tự động lưu khi session ERROR (lỗi)
- Tự động lưu khi session STOPPED (dừng thủ công)
- Tránh lưu trùng bằng session key tracking

### Real-time sync
- Tự động cập nhật khi có bản ghi mới
- Multi-device sync (nhiều thiết bị cùng xem)

### Local + Firebase dual storage
- Local history: lưu trong browser (localStorage)
- Firebase history: lưu trên cloud (Firestore)
- Toggle giữa hai chế độ dễ dàng

### Test functionality
- Test save button để kiểm tra kết nối
- Stats cards để xem tổng quan
- Filter và search dễ dàng

## 🎯 Workflow hoạt động

```
1. User chạy demo bơm
   ↓
2. PumpSimulationHook tracks state
   ↓
3. Khi state → DONE/ERROR/STOPPED
   ↓
4. usePumpSimulationWithFirebase phát hiện thay đổi
   ↓
5. Tạo PumpHistoryRecord
   ↓
6. Gọi savePumpHistory()
   ↓
7. addDoc() vào Firestore collection 'pump_history'
   ↓
8. Real-time listener tự động cập nhật UI
```

## 🐛 Known Issues / Limitations

1. **Firebase Free Tier Limits:**
   - 50K reads/day
   - 20K writes/day
   - 1GB storage

2. **Firestore Security Rules:**
   - Mặc định ở Test mode (30 ngày)
   - Cần cấu hình rules cho production

3. **Device ID:**
   - Được lưu trong localStorage
   - Sẽ thay đổi nếu user xóa browser data
   - Không sync giữa các browser

4. **Real-time sync:**
   - Cần internet connection
   - Có độ trễ nhỏ (100-500ms)
